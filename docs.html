06 Variables & Types

A variable is a named container that holds a value in memory. Think of it like a labeled box — you put something in it, give it a name, and refer to it later by that name. Use let for values that will change, and const for values that must stay fixed.

HOLEXA is statically typed — every variable has a fixed type that does not change. However, HOLEXA uses type inference — you usually don't need to write the type, the compiler figures it out from the value you assign.

The 5 basic types

TypeWhat it storesExample valueNotes
IntWhole numbers — no decimals42, -10, 064-bit signed integer
FloatDecimal numbers3.14, -0.5, 100.064-bit floating point
StringText of any length"hello", "HOLEXA"Always in double quotes
BoolOnly two values possibletrue or falseUsed in conditions
NoneNo value / emptynoneLike null in other languages
variables.hlxholexa
fn main() {
    // ── Declaring variables with let ──────────────
    let name    = "Dragon"    // Type: String — text
    let age     = 25         // Type: Int — whole number
    let height  = 5.9        // Type: Float — decimal
    let active  = true       // Type: Bool — true or false
    const MAX   = 100        // Type: Int — can NEVER change

    // ── Printing values ───────────────────────────
    // String: print directly
    print("Name:   " + name)

    // Numbers and Bool: must convert to string with str()
    print("Age:    " + str(age))
    print("Height: " + str(height))
    print("Active: " + str(active))

    // ── Changing a let variable ───────────────────
    age = age + 1    // OK — let can change
    print("Next year age: " + str(age))

    // ── Math operators ────────────────────────────
    print(str(10 + 5))     // 15  — addition
    print(str(10 - 3))     // 7   — subtraction
    print(str(4  * 5))     // 20  — multiplication
    print(str(10 / 3))     // 3   — integer division (no decimals)
    print(str(10 % 3))     // 1   — remainder (modulo)
    print(str(2  ** 10))   // 1024 — power (2 to the 10th)

    // ── String joining (concatenation) ───────────
    let greeting = "Hello, " + name + "!"
    print(greeting)          // Hello, Dragon!
}
Output
Name: Dragon Age: 25 Height: 5.9 Active: true Next year age: 26 15 7 20 3 1 1024 Hello, Dragon!
⚠️ const cannot be changed. Writing MAX = 200 after declaring const MAX = 100 will cause a compile error. This is intentional — it prevents accidental changes to important values.
💡 To print a number, always wrap it in str(). Use print(str(42)) — not print(42). Numbers and strings cannot be mixed without conversion.

07 User Input

The input() function lets your program communicate with the user. It shows a message, waits for the user to type something and press Enter, then gives you back what they typed as a String.

Critical rule: input() ALWAYS returns a String — even if the user types a number like 42, you get the text "42". To use it as a number, you must convert it with int() or float().

PatternReturnsWhen to use
input("prompt")StringReading any text — names, answers, commands
int(input("prompt"))IntReading a whole number like age, score, count
float(input("prompt"))FloatReading a decimal like price, temperature, weight
input_demo.hlxholexa
fn main() {
    // ── Text input ────────────────────────────────
    let name = input("Enter your name: ")
    print("Hello " + name + "! Welcome to HOLEXA.")

    // ── Number input ──────────────────────────────
    // input() returns String "25" — int() converts it to 25
    let age = int(input("How old are you? "))

    if age >= 18 {
        print("You are an adult!")
    } else {
        print("You are " + str(18 - age) + " years from 18.")
    }

    // ── Calculator using input ────────────────────
    print("--- Mini Calculator ---")
    let a = int(input("First number:  "))
    let b = int(input("Second number: "))
    print(str(a) + " + " + str(b) + " = " + str(a + b))
    print(str(a) + " * " + str(b) + " = " + str(a * b))
}
Output (user typed: Dragon, 15, 10, 4)
Enter your name: Dragon Hello Dragon! Welcome to HOLEXA. How old are you? 15 You are 3 years from 18. --- Mini Calculator --- First number: 10 Second number: 4 10 + 4 = 14 10 * 4 = 40
⚠️ input() ALWAYS returns String. If the user types 42, you get "42" — the text, not the number. Without int(), doing math on it will fail. This is a very common beginner mistake.

08 Functions

A function is a named, reusable block of code that performs one specific job. Instead of writing the same code multiple times, you write it once as a function and call it whenever needed. Functions make programs shorter, cleaner, and much easier to fix when something goes wrong.

Function anatomy — understanding each part

Return type-> StringWhat type of value the function sends back

functions.hlxholexa
// Takes a String, returns a String
fn greet(name: String) -> String {
    return "Hello " + name + "!"
}

// Takes two Ints, returns their sum as Int
fn add(a: Int, b: Int) -> Int {
    return a + b
}

// Returns Bool — true if number is even, false if odd
fn is_even(n: Int) -> Bool {
    if n % 2 == 0 { return true }
    return false
}

// No return type — just does something (prints)
fn say_hello() {
    print("Hello from HOLEXA!")
}

// Three parameters — returns the largest one
fn biggest(a: Int, b: Int, c: Int) -> Int {
    if a >= b and a >= c { return a }
    if b >= a and b >= c { return b }
    return c
}

fn main() {
    // Call greet() with "Dragon" — gets "Hello Dragon!" back
    print(greet("Dragon"))

    // add() returns 30 — wrap in str() to print it
    print(str(add(10, 20)))

    print(str(is_even(4)))       // true
    print(str(is_even(7)))       // false
    say_hello()
    print(str(biggest(5, 12, 8))) // 12
}
Output
Hello Dragon! 30 true false Hello from HOLEXA! 12
💡 Rule: Always define functions before main() — or at least before the line where you first call them.

09 Control Flow

Control flow decides which parts of your code run and when. Without it, every line would always execute in order. With if, else if, and else, you can make your program respond differently to different situations.

control.hlxholexa
// HOLEXA checks conditions top to bottom.
// As soon as one is true, it runs that block and stops.
fn get_grade(score: Int) -> String {
    if score >= 90      { return "A+ — Excellent!"  }
    else if score >= 80 { return "A  — Great job!"  }
    else if score >= 70 { return "B  — Good"        }
    else if score >= 60 { return "C  — Average"     }
    else                 { return "F  — Try again"   }
    // else runs only if ALL above conditions were false
}

fn main() {
    print(get_grade(95))    // A+ — Excellent!
    print(get_grade(75))    // B  — Good
    print(get_grade(45))    // F  — Try again

    // ── Combining conditions ──────────────────────
    let x = 42

    // and — BOTH must be true
    if x > 10 and x < 100 {
        print("x is between 10 and 100")
    }

    // or — AT LEAST ONE must be true
    if x < 0 or x > 1000 {
        print("out of range")
    } else {
        print("x is in range")
    }

    // not — reverses the condition
    if not (x == 0) { print("x is not zero") }
}
Output
A+ — Excellent! B — Good F — Try again x is between 10 and 100 x is in range x is not zero

10 Loops

Loops let you repeat code without writing it over and over. HOLEXA has two types of loops: for (when you know exactly how many times) and while (when you want to keep going until a condition changes).

loops.hlxholexa
fn main() {
    // ── for loop ──────────────────────────────────
    // 0..5 means: start at 0, go UP TO (not including) 5
    // So this runs for i = 0, 1, 2, 3, 4
    print("Counting:")
    for i in 0..5 {
        print("  i = " + str(i))
    }

    // ── while loop ────────────────────────────────
    // Keeps running as long as the condition is true
    print("Countdown:")
    let n = 3
    while n > 0 {
        print("  " + str(n) + "...")
        n = n - 1    // ← MUST update n or loop runs forever!
    }
    print("  Blast off! 🚀")

    // ── Practical: sum numbers ────────────────────
    let total = 0
    for i in 1..101 { total = total + i }  // sum 1 to 100
    print("Sum 1 to 100 = " + str(total))

    // ── Nested loops: multiplication table ───────
    for i in 1..4 {
        for j in 1..4 {
            print(str(i) + " × " + str(j) + " = " + str(i*j))
        }
    }
}
Output
Counting: i = 0 i = 1 i = 2 i = 3 i = 4 Countdown: 3... 2... 1... Blast off! 🚀 Sum 1 to 100 = 5050 1 × 1 = 1 1 × 2 = 2 ... 3 × 3 = 9
⚠️ Infinite loop danger! In a while loop, always make sure the variable in the condition eventually changes — otherwise the condition never becomes false and the loop runs forever, freezing your program.

11 Classes & Objects

A class is a blueprint — like a recipe or a cookie cutter. An object is what you create from that blueprint — like the actual cookie. Classes group related data (fields) and related actions (methods) together into one unit. This is called Object-Oriented Programming (OOP).

For example, a Dragon class might have fields like name and power, and methods like roar() and fly().

classes.hlxholexa
// Define the blueprint (class)
class Dragon {
    // Fields — data each Dragon object holds
    name:  String
    power: Int
    alive: Bool

    // Constructor — creates a new Dragon object
    // Called like: Dragon.new("HOLEXA", 9999)
    fn new(name: String, power: Int) -> Dragon {
        return Dragon {
            name:  name,
            power: power,
            alive: true      // all dragons start alive
        }
    }

    // Method — self means "the current Dragon object"
    // self.name accesses THIS dragon's name field
    fn roar(self) -> String {
        return self.name + " ROARS! Power = " + str(self.power)
    }

    fn is_legendary(self) -> Bool {
        return self.power > 9000
    }

    fn describe(self) -> String {
        return "Dragon[" + self.name + ", power=" + str(self.power) + "]"
    }
}

fn main() {
    // Create two different Dragon objects from the same class
    let d1 = Dragon.new("HOLEXA", 9999)
    let d2 = Dragon.new("Fire",   5000)

    // Call methods on each object
    print(d1.roar())
    print(d2.roar())
    print(d1.describe())
    print("d1 legendary: " + str(d1.is_legendary()))
    print("d2 legendary: " + str(d2.is_legendary()))
}
Output
HOLEXA ROARS! Power = 9999 Fire ROARS! Power = 5000 Dragon[HOLEXA, power=9999] d1 legendary: true d2 legendary: false
ℹ️ Key insight: d1 and d2 are two separate objects created from the same Dragon blueprint. Changing d1.power does not affect d2.power — they are completely independent.

12 Error Handling

Errors happen in every program. A file might not exist. A number might be zero when you try to divide by it. Network might be down. In most languages, unhandled errors crash the program. HOLEXA uses try and catch to handle errors gracefully — the program continues running.

The rule is simple: put risky code inside a try block. If an error happens, execution jumps to the catch block. If no error happens, catch is skipped.

errors.hlxholexa
fn divide(a: Int, b: Int) -> Int {
    // Check for the error BEFORE it happens
    if b == 0 {
        print("[ERROR] Cannot divide by zero!")
        return 0    // return safe default value
    }
    return a / b
}

fn main() {
    // try block — code that might fail goes here
    try {
        print("10 / 2 = " + str(divide(10, 2)))  // works fine
        print("10 / 0 = " + str(divide(10, 0)))  // error handled inside
        print("Code continues after error!")
    } catch err {
        // Only runs if something threw an error inside try
        print("Caught error: " + err)
    }

    // This always runs — program did NOT crash
    print("Program finished successfully!")
}
Output
10 / 2 = 5 [ERROR] Cannot divide by zero! 10 / 0 = 0 Code continues after error! Program finished successfully!
Without try/catchWith try/catch
Error → Program crashes → User loses dataError → Caught → Program keeps running safely
No message to userYou can print a helpful error message
Hard to debugEasy to find which operation failed

14 Recursion

Recursion is when a function calls itself. It is a powerful technique for problems that can be broken into smaller versions of the same problem. Every recursive function needs two things:

1. A base case — the condition that stops the recursion (otherwise it runs forever).
2. A recursive case — calling the function with a smaller or simpler input.

recursion.hlxholexa
// factorial(5) = 5 × 4 × 3 × 2 × 1 = 120
fn factorial(n: Int) -> Int {
    if n <= 1 { return 1 }           // ← base case: STOP here
    return n * factorial(n - 1)   // ← recursive: calls itself
}
// How it works for factorial(4):
// factorial(4) → 4 × factorial(3)
// factorial(3) → 3 × factorial(2)
// factorial(2) → 2 × factorial(1)
// factorial(1) → 1  (base case!)
// Result: 4 × 3 × 2 × 1 = 24

// fibonacci sequence: 0, 1, 1, 2, 3, 5, 8, 13...
fn fibonacci(n: Int) -> Int {
    if n <= 1 { return n }            // base case
    return fibonacci(n - 1) + fibonacci(n - 2)
}

fn main() {
    print("Factorials:")
    for i in 1..8 {

fn save_history(expr: String, result: Float) {
    let entry = '{"expr":"' + expr + '","result":'

    fs.append("history.txt", entry + "\n")

fn main() {
    print("=== HOLEXA Scientific Calculator ===")
    print("Date: " + time.date())

        } else if cmd == "pow" {
            let b = float(input("Base: "))
            let e = float(input("Exponent: "))
            let r = math.pow(b, e)
            print(str(b) + "^" + str(e) + " = " + str(r))
            save_history(str(b) + "^" + str(e), r)

        } else if cmd == "circle" {
            let r = float(input("Radius: "))
            let area = math.PI * r * r
            print("Area = " + str(area))

        }

Basic Bridge — Run HOLEXA from Python

bridge.py
import subprocess, tempfile, os

def run_holexa(code: str) -> str:
    """Run any HOLEXA code and return its output"""
    with tempfile.NamedTemporaryFile(suffix='.hlx', mode='w', delete=False) as f:
        f.write(code)
        fname = f.name
    try:
        result = subprocess.run(
            ['hlxc', 'run', fname],
            capture_output=True, text=True, timeout=30
        )
        return result.stdout if result.returncode == 0 else result.stderr
    finally:
        os.unlink(fname)

# Example: use HOLEXA to compute fibonacci
output = run_holexa('''

fn ask_ai(question: String, api_key: String) -> String {
    // Set headers for OpenAI

    http.set_header("Content-Type", "application/json")

    // Build request body

    // Call the API

}

    }
}

fn is_prime(n: Int) -> Bool {
    if n < 2 { return false }
    if n == 2 { return true }
    if n % 2 == 0 { return false }

        if n % i == 0 { return false }
        i = i + 2
    }
    return true
